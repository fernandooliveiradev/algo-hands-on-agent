# AHO Guided Lessons

## Quantidade de prática

A prática durante o módulo é adaptativa. Não imponha cinco exercícios a cada subtópico.

## Checkpoint final

Gere cinco evidências distintas:

1. `direct_application`
2. `independent_application`
3. `integration`
4. `diagnosis`
5. `explanation_transfer`

## Contagem

Uma evidência só satisfaz domínio quando:

- resultado `correct`;
- sem dica relevante;
- nota pelo menos 0,8;
- competência coerente com o módulo.

## Remediação

- erro conceitual: reduzir o problema e voltar ao pré-requisito;
- erro de procedimento: teste de mesa e nova tentativa;
- erro de sintaxe: correção mínima sem rebaixar domínio conceitual;
- acerto com dica: nova variação independente;
- acerto sem explicação: pedir explicação.

## Restrições por módulo

- Módulo 1 (Pensamento Computacional): NUNCA use pseudocódigo, fluxogramas, teste de mesa, código ou termos técnicos de programação.
- Módulo 2 (Algoritmos): NUNCA use sintaxe Python. Apenas linguagem natural, pseudocódigo, fluxogramas e teste de mesa.
- Módulo 3 em diante: Python é permitido, mas não introduza conceitos de módulos futuros.

Não afirme avanço; informe que a evidência foi registrada e deixe a aplicação confirmar o módulo atual.
