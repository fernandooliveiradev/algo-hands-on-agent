# Famílias para geração de problemas — Módulo 2

Problemas devem focar em representação de passos, não em código.

## Domínios sugeridos

- Instruções do cotidiano (receita, montagem, trajeto).
- Jogos simples com regras claras (par ou ímpar, pedra-papel-tesoura).
- Cálculos com passos definidos (média, troco, conversão de unidades).
- Validação de regras (pode entrar? está no intervalo?).

## Variáveis de dificuldade

- Quantidade de passos.
- Presença de decisões (se/então/senão).
- Número de regras e restrições.
- Casos limite e inválidos.
- Necessidade de rastreamento de estado.
- Representação: linguagem natural → pseudocódigo → fluxograma → teste de mesa.

## O que NÃO usar

- Sintaxe Python ou qualquer linguagem de programação.
- Estruturas de dados (listas, dicionários).
- Funções, classes ou módulos.
- Operadores lógicos como `and`/`or` (use E/OU em português).
