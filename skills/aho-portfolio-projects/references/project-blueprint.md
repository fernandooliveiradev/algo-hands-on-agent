# Blueprint de projeto

Todo projeto deve conter:

- problema;
- público ou usuário;
- escopo mínimo;
- requisitos funcionais;
- dados;
- regras de negócio;
- marcos;
- testes;
- instruções de execução;
- decisões técnicas;
- melhorias futuras.

Variáveis de dificuldade: persistência, número de entidades, integração externa, autenticação, concorrência e publicação.
